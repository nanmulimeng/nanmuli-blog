# 第九轮项目收尾检查计划

> 最终收尾检查 | 2026-04-06

## 一、检查目标

对全部48个文件进行最终收尾检查，确保：
1. **整体一致性**：前后端接口、命名、逻辑一致
2. **部署就绪性**：配置正确，文档完整
3. **潜在问题排查**：边界情况、竞态条件、内存泄漏
4. **代码整洁度**：无冗余代码，符合规范

## 二、模块分派

### 模块A：后端整体一致性检查
**代理**: `final-check-backend`
**检查范围**：后端所有修改文件（14个）

检查清单：
1. **接口一致性**
   - Controller路径与前端调用是否一致
   - 请求/响应DTO字段是否匹配
   - 错误码前后端是否统一

2. **事务一致性**
   - @Transactional使用是否正确
   - 跨聚合操作是否完整
   - 缓存与DB一致性

3. **安全一致性**
   - 所有接口权限控制是否到位
   - Sa-Token配置与代码是否一致
   - 敏感操作是否有日志记录

4. **配置一致性**
   - application.yml与application-prod.yml差异
   - 数据库迁移脚本是否完整

---

### 模块B：前端整体一致性检查
**代理**: `final-check-frontend`
**检查范围**：前端所有修改文件（28个）

检查清单：
1. **API调用一致性**
   - 所有API调用路径与后端一致
   - 请求参数、响应数据处理正确
   - 错误处理统一

2. **状态管理一致性**
   - Pinia状态与组件使用一致
   - 登录状态同步正确
   - 缓存策略一致

3. **路由一致性**
   - 路由定义与实际链接一致
   - 权限路由控制正确
   - 面包屑与导航匹配

4. **样式一致性**
   - 主题变量使用一致
   - 暗色/亮色模式切换无遗漏
   - 响应式断点统一

---

### 模块C：部署就绪性检查
**代理**: `final-check-deployment`
**检查范围**：配置文件、构建配置、文档

检查清单：
1. **后端部署配置**
   - application-prod.yml配置完整
   - 数据库连接配置正确
   - 日志配置合理
   - JVM参数建议

2. **前端部署配置**
   - vite.config.ts生产配置
   - 环境变量配置
   - 构建输出配置

3. **数据库脚本**
   - init.sql完整性
   - 迁移脚本顺序正确
   - 索引创建完整

4. **文档完整性**
   - README.md部署说明
   - 配置说明完整
   - 常见问题排查

---

### 模块D：潜在问题深度排查
**代理**: `final-check-issues`
**检查范围**：全项目潜在问题

检查清单：
1. **竞态条件**
   - 异步操作状态更新安全
   - 请求取消是否正确
   - 防抖节流是否到位

2. **内存泄漏**
   - 定时器是否清理
   - 事件监听是否移除
   - 闭包引用是否释放

3. **边界情况**
   - 空数据处理
   - 超长内容处理
   - 网络异常处理

4. **性能问题**
   - 大数据列表渲染
   - 图片懒加载
   - 代码分割

---

### 模块E：代码整洁度与规范检查
**代理**: `final-check-quality`
**检查范围**：全项目代码规范

检查清单：
1. **代码规范**
   - 命名规范一致
   - 注释完整性
   - 类型定义完整

2. **冗余代码**
   - 未使用导入
   - 死代码
   - 重复逻辑

3. **TODO/FIXME**
   - 检查所有标记
   - 评估是否需要处理

4. **日志规范**
   - 敏感信息不打印
   - 日志级别合理
   - 关键操作有日志

---
